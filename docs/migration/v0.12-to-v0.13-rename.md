# Migration Guide: v0.12 to v0.13 (Binary Rename)

This guide covers migrating from v0.12 (using `agentic-memorizer`) to v0.13 (using `memorizer`).

## Breaking Changes

**Version v0.13.0** introduces the following breaking changes:

- **Binary name**: `agentic-memorizer` → `memorizer`
- **App directory**: `~/.agentic-memorizer` → `~/.memorizer`
- **Service names**:
  - systemd: `agentic-memorizer.service` → `memorizer.service`
  - launchd: `com.<user>.agentic-memorizer` → `com.<user>.memorizer`
  - supervisor: `agentic-memorizer.conf` → `memorizer.conf`
- **MCP server name**: `agentic-memorizer` → `memorizer`
- **Integration adapter versions**: All integration adapters bumped to v2.0.0 due to breaking changes

**What stays the same:**
- Go module path: `github.com/leefowlercu/agentic-memorizer`
- Project branding: "Agentic Memorizer"
- Environment variable prefix: `MEMORIZER_*`
- Configuration file format and settings

## Migration Steps

### Step 1: Stop the Old Daemon

**systemd (Linux):**
```bash
# Stop and disable the old service
systemctl --user stop agentic-memorizer
systemctl --user disable agentic-memorizer
```

**launchd (macOS):**
```bash
# Stop and unload the old service
launchctl stop com.$(whoami).agentic-memorizer
launchctl unload ~/Library/LaunchAgents/com.$(whoami).agentic-memorizer.plist
```

**supervisor:**
```bash
# Stop the old service
sudo supervisorctl stop agentic-memorizer
```

**Direct daemon (no service manager):**
```bash
# Stop the daemon
agentic-memorizer daemon stop
```

### Step 2: Remove Old Integrations

**Remove all configured integrations** before upgrading to avoid old binary path issues:

```bash
# List current integrations
agentic-memorizer integrations list

# Remove each integration (example: claude-code-hook)
agentic-memorizer integrations remove claude-code-hook

# Remove other integrations if configured
agentic-memorizer integrations remove claude-code-mcp
agentic-memorizer integrations remove gemini-cli-mcp
agentic-memorizer integrations remove codex-cli-mcp
```

**Important**: Integration adapters v2.0.0 detect and reject old binary paths. Removing integrations before upgrading ensures a clean migration.

### Step 3: Back Up Your Data (Optional)

If you want to preserve your current setup while testing the new version:

```bash
# Back up app directory
cp -r ~/.agentic-memorizer ~/.agentic-memorizer.backup

# Back up old binary (if installed via make install)
cp ~/.local/bin/agentic-memorizer ~/.local/bin/agentic-memorizer.backup
```

### Step 4: Install New Binary

**Option A: Using go install (recommended):**
```bash
go install github.com/leefowlercu/agentic-memorizer@latest
```

This installs `memorizer` to `~/go/bin/memorizer`.

**Option B: Build from source:**
```bash
git clone https://github.com/leefowlercu/agentic-memorizer.git
cd agentic-memorizer
make install
```

This installs `memorizer` to `~/.local/bin/memorizer`.

**Verify installation:**
```bash
memorizer version
```

You should see version v0.13.0 or higher.

### Step 5: Migrate App Directory

**Option A: Move existing data (recommended):**
```bash
# Move entire app directory to new location
mv ~/.agentic-memorizer ~/.memorizer
```

**Option B: Fresh start (alternative):**
```bash
# Initialize new app directory
memorizer initialize

# Manually copy memory files
cp -r ~/.agentic-memorizer/memory/* ~/.memorizer/memory/

# Copy config if customized
cp ~/.agentic-memorizer/config.yaml ~/.memorizer/config.yaml
```

### Step 6: Update Configuration (If Needed)

**Check and update skip_files:**
```bash
# Edit config
vi ~/.memorizer/config.yaml

# Change skip_files from:
skip_files: [agentic-memorizer]

# To:
skip_files: [memorizer]
```

**Note**: The `initialize` command creates config with correct defaults for v0.13.

### Step 7: Start FalkorDB

```bash
# Start graph database
memorizer graph start

# Verify status
memorizer graph status
```

### Step 8: Start New Daemon

**Direct daemon (foreground):**
```bash
# Start daemon directly (Ctrl+C to stop)
memorizer daemon start
```

**As a system service (recommended):**

For production use, set up the daemon as a system service. See the "Running as a Service" section in the README for complete systemd (Linux) and launchd (macOS) setup instructions with inline configuration templates.

Alternatively, run `memorizer initialize` to use the interactive wizard which can automatically install system services.

### Step 9: Verify Daemon Health

```bash
# Check daemon status
memorizer daemon status

# Check logs
memorizer daemon logs

# Verify health endpoint (if HTTP server enabled)
curl http://localhost:8080/health
```

### Step 10: Re-setup Integrations

**Set up each integration you were using:**

```bash
# List available integrations
memorizer integrations list

# Setup Claude Code hook integration
memorizer integrations setup claude-code-hook

# Setup Claude Code MCP integration
memorizer integrations setup claude-code-mcp

# Setup Gemini CLI MCP integration (if used)
memorizer integrations setup gemini-cli-mcp

# Setup Codex CLI MCP integration (if used)
memorizer integrations setup codex-cli-mcp
```

**Verify integrations:**
```bash
# Check integration status
memorizer integrations status

# Health check
memorizer integrations health
```

### Step 11: Test Integration

**For Claude Code:**
1. Start a new Claude Code session
2. Verify the index loads automatically (SessionStart hook)
3. Check that Claude can read memory files

**For MCP integrations:**
```bash
# Test MCP server manually
memorizer mcp start

# In another terminal, use MCP tools
# (requires MCP client like Claude Desktop or Gemini CLI)
```

### Step 12: Clean Up Old Files (Optional)

**After verifying everything works, remove old files:**

```bash
# Remove old binary
rm ~/.local/bin/agentic-memorizer  # If installed via make install
rm ~/go/bin/agentic-memorizer      # If installed via go install

# Remove old service files
rm ~/.config/systemd/user/agentic-memorizer.service  # Linux
rm ~/Library/LaunchAgents/com.$(whoami).agentic-memorizer.plist  # macOS
sudo rm /etc/supervisor/conf.d/agentic-memorizer.conf  # supervisor

# Remove backup if no longer needed
rm -rf ~/.agentic-memorizer.backup
```

## Troubleshooting

### Old Binary Path Detected

**Error**: Integration adapter reports old binary path in config.

**Solution**: Remove and re-setup the integration:
```bash
memorizer integrations remove <integration-name>
memorizer integrations setup <integration-name>
```

### Config Validation Fails

**Error**: `memorizer config validate` reports validation errors.

**Solution**: Check for old paths in config:
```bash
# Check config
cat ~/.memorizer/config.yaml

# Look for old paths like ~/.agentic-memorizer
# Update to ~/.memorizer
```

### Daemon Won't Start

**Error**: Daemon fails to start after migration.

**Symptoms**: Port conflict, missing FalkorDB, permission errors.

**Solutions**:
1. **Port conflict**: Check if old daemon still running
   ```bash
   ps aux | grep memorizer
   killall agentic-memorizer  # Kill old daemon
   ```

2. **FalkorDB not running**:
   ```bash
   memorizer graph start
   memorizer graph status
   ```

3. **Permission errors**:
   ```bash
   # Verify ownership
   ls -la ~/.memorizer

   # Fix if needed
   chown -R $(whoami) ~/.memorizer
   ```

### Index Not Loading in Claude Code

**Error**: SessionStart hook not firing or index not appearing.

**Solutions**:
1. **Verify hook is installed**:
   ```bash
   memorizer integrations status claude-code-hook
   ```

2. **Check hook file**:
   ```bash
   cat ~/.claude/sessionStart.json
   # Should reference memorizer, not agentic-memorizer
   ```

3. **Re-setup integration**:
   ```bash
   memorizer integrations remove claude-code-hook
   memorizer integrations setup claude-code-hook
   ```

### MCP Server Not Found

**Error**: MCP client can't find `memorizer` server.

**Solutions**:
1. **Check MCP config** (Claude Desktop example):
   ```bash
   cat ~/Library/Application\ Support/Claude/claude_desktop_config.json

   # Should have:
   # "memorizer": {
   #   "command": "/path/to/memorizer",
   #   "args": ["mcp", "start"]
   # }
   ```

2. **Verify binary path**:
   ```bash
   which memorizer
   # Should show ~/go/bin/memorizer or ~/.local/bin/memorizer
   ```

3. **Re-setup integration**:
   ```bash
   memorizer integrations remove claude-code-mcp
   memorizer integrations setup claude-code-mcp
   ```

## Rollback Procedure

If you need to rollback to v0.12:

### Step 1: Stop New Daemon
```bash
memorizer daemon stop
# Or stop via service manager
```

### Step 2: Restore Old Binary
```bash
# If you kept backup
mv ~/.local/bin/agentic-memorizer.backup ~/.local/bin/agentic-memorizer

# Or reinstall v0.12
go install github.com/leefowlercu/agentic-memorizer@v0.12.1
```

### Step 3: Restore App Directory
```bash
# If you kept backup
mv ~/.memorizer ~/.memorizer.v013.backup
mv ~/.agentic-memorizer.backup ~/.agentic-memorizer
```

### Step 4: Restore Service Files
```bash
# Restore service files from backup if you made one
# Or manually update the service file paths to use the old binary
```

### Step 5: Start Old Daemon
```bash
agentic-memorizer daemon start
# Or start via service manager if configured
```

### Step 6: Restore Integrations
```bash
# Re-setup with old binary
agentic-memorizer integrations setup claude-code-hook
# etc.
```
